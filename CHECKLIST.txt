Siga esta lista passo a passo para ativar o CI/CD:

┌─────────────────────────────────────────────────────────────┐
│  PARTE 1: PREPARAÇÃO LOCAL (5 minutos)                     │
└─────────────────────────────────────────────────────────────┘

[ ] 1. Abrir terminal no projeto
    cd /Users/Projetos/dogCeo

[ ] 2. Verificar configuração (opcional mas recomendado)
    ./verificar-config.sh

[ ] 3. Adicionar todos os arquivos ao Git
    git add .

[ ] 4. Fazer commit
    git commit -m "feat: Adicionar CI/CD completo"

[ ] 5. Verificar status
    git status
    (Deve mostrar: "nothing to commit, working tree clean")

┌─────────────────────────────────────────────────────────────┐
│  PARTE 2: CRIAR REPOSITÓRIO NO GITHUB (2 minutos)          │
└─────────────────────────────────────────────────────────────┘

[ ] 6. Acessar https://github.com/new

[ ] 7. Preencher:
    Nome: dogCeo (ou outro nome)
    Descrição: Testes automatizados da API Dog CEO
    Público/Privado: Sua escolha

[ ] 8. NÃO MARCAR "Initialize with README"
    (Já temos um README!)

[ ] 9. Clicar em "Create repository"

[ ] 10. Copiar a URL do repositório
     Exemplo: https://github.com/SEU_USUARIO/dogCeo.git

┌─────────────────────────────────────────────────────────────┐
│  PARTE 3: CONECTAR E FAZER PUSH (2 minutos)                │
└─────────────────────────────────────────────────────────────┘

[ ] 11. Adicionar remote (substitua SEU_USUARIO)
     git remote add origin https://github.com/SEU_USUARIO/dogCeo.git

[ ] 12. Verificar remote
     git remote -v
     (Deve mostrar origin com sua URL)

[ ] 13. Renomear branch para main (se necessário)
     git branch -M main

[ ] 14. Fazer push
     git push -u origin main

[ ] 15. Aguardar push concluir
     (Pode pedir senha - use Personal Access Token se necessário)

┌─────────────────────────────────────────────────────────────┐
│  PARTE 4: CONFIGURAR GITHUB ACTIONS (3 minutos)            │
└─────────────────────────────────────────────────────────────┘

[ ] 16. Ir ao repositório no GitHub
     https://github.com/SEU_USUARIO/dogCeo

[ ] 17. Clicar na aba "Actions"
     Você deve ver o workflow "API Tests CI" rodando

[ ] 18. Aguardar execução terminar (2-5 minutos)
     ✅ Verde = Sucesso
     ❌ Vermelho = Ver logs para debug

[ ] 19. Ir em Settings → Actions → General

[ ] 20. Rolar até "Workflow permissions"

[ ] 21. Selecionar "Read and write permissions"

[ ] 22. Marcar "Allow GitHub Actions to create and approve pull requests"

[ ] 23. Clicar em "Save"

┌─────────────────────────────────────────────────────────────┐
│  PARTE 5: CONFIGURAR GITHUB PAGES (2 minutos)              │
└─────────────────────────────────────────────────────────────┘

[ ] 24. Verificar que a branch gh-pages foi criada
     (No GitHub: Botão de branches, procurar "gh-pages")

[ ] 25. Ir em Settings → Pages

[ ] 26. Em "Source", selecionar:
     Branch: gh-pages
     Folder: / (root)

[ ] 27. Clicar em "Save"

[ ] 28. Aguardar 5-10 minutos

[ ] 29. Copiar o link que aparece (algo como):
     https://SEU_USUARIO.github.io/dogCeo/

[ ] 30. Acessar o link para ver o relatório Allure!

┌─────────────────────────────────────────────────────────────┐
│  PARTE 6: ATUALIZAR README (1 minuto - OPCIONAL)           │
└─────────────────────────────────────────────────────────────┘

[ ] 31. Abrir README.md

[ ] 32. Substituir todas as ocorrências de "USUARIO" pelo seu usuário

[ ] 33. Salvar e fazer commit/push:
     git add README.md
     git commit -m "docs: Atualizar badges com usuário correto"
     git push

┌─────────────────────────────────────────────────────────────┐
│  ✅ VERIFICAÇÃO FINAL                                       │
└─────────────────────────────────────────────────────────────┘

Verifique se tudo está funcionando:

[ ] GitHub Actions mostra builds com sucesso ✅

[ ] Relatório Allure acessível via GitHub Pages
    https://SEU_USUARIO.github.io/dogCeo/

[ ] Artefatos disponíveis para download
    (Actions → Execução → Artifacts)

[ ] Badges no README funcionando
    (Devem mostrar status verde)

[ ] Workflow agendado configurado
    (Actions → Testes Agendados → Habilitado)
